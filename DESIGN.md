# ระบบการออกแบบ: BBP (บลูม บลูม พอ)

## ภาพรวม

**ดาวเหนือสร้างสรรค์: "สถานที่ศักดิ์สิทธิ์ของการดำเนินงาน"**

BBP ออกแบบมาเป็นพื้นที่ทำงานมืออาชีพที่พนักงานร้านสามารถทำงานได้อย่างมั่นใจและชัดเจน ทุกองค์ประกอบให้บริการงาน—ไม่มีการตกแต่งให้สวยงาม ไม่มีความยุ่งเหยิง อินเตอร์เฟซมีความสงบ เป็นระเบียบ และสามารถคาดเดาได้ ช่วยให้ผู้ใช้มุ่งเน้นไปที่การบริหารธุรกิจมากกว่าการเรียนรู้เครื่องมือ

พาเลตมีความอบอุ่นและประกอบด้วย: พื้นหลังธรรมชาติที่มีโทนสีสำหรับพื้นผิว สมดุลด้วยสีเขียวเซจจริงใจสำหรับการกระทำหลัก แอมเบอร์เน้นมีความรักษาอักษรและจริงใจ การพิมพ์ชัดเจนและมีประสิทธิภาพ พร้อมระยะห่างอุดมสมบูรณ์ที่เคารพความสนใจของผู้ใช้

ระบบนี้ปฏิเสธชัดเจนแบรนด์สัตว์เลี้ยงน่ารัก (มาสคอต สีพาสเทลที่เล่นสนุก เต็มไปด้วยอีโมจิ) และความซ้ำซากขององค์กร (สีเทาเดียว ความเย็นของ "ซอฟต์แวร์องค์กร") โทนเป็นมืออาชีพแต่เป็นมนุษย์—คู่ค้าที่สามารถทำได้ ไม่ใช่เครื่องจักร

**ลักษณะสำคัญ:**
- โครงร่างสะอาด ปราศจากสิ่งรบกวน
- นิวทรัลอบอุ่นพร้อมสีเขียวเซจหลักที่เชื่อมั่น
- ความคมชัดสูงเพื่อความสามารถในการอ่านภายใต้การส่องสว่างทั่วไปของธุรกิจ
- ลำดับชั้นข้อมูลที่มีประสิทธิภาพซึ่งเคารพความสนใจของผู้ใช้
- การเข้าถึงได้ก่อน (มาตรฐาน WCAG 2.1 AAA)
- การรวมภาษาไทยเป็นหลัก ไม่ใช่สิ่งที่พิจารณาภายหลัง

## สี: พาเลตเซจและแอมเบอร์

พาเลตสร้างขึ้นจากการปฏิเสธมืออาชีพพร้อมโทนสีอบอุ่นและลึกซึ้ง สีเขียวหลัก (Fresh Sage) ครอบครอง ~8–12% ของอินเตอร์เฟซและส่งสัญญาณการกระทำหลัก แอมเบอร์เน้น (Warm Amber) มีความรักษาอักษรมากขึ้น ใช้สำหรับไฮไลท์ การเตือน และเน้นรองพื้น นิวทรัลเป็นสีอบอุ่น (ไม่เคยสีเทาบริสุทธิ์) เพื่อให้อินเตอร์เฟซที่ทำเทคโนโลยีอ่อนลง

### สีหลัก

- **Fresh Sage** (`oklch(0.55 0.15 145)`): ปุ่มการกระทำหลัก การนำทางที่ใช้งาน ตัวบ่งชี้โฟกัส ลิงก์ นี่คือสีแห่งความมั่นใจและความสำเร็จ ความอิ่มตัวแบบปานกลางและแสงกลางทำให้มันทำงานได้ภายใต้สภาพแสงที่แตกต่างกัน

### เน้น

- **Warm Amber** (`oklch(0.85 0.08 60)`): ไฮไลท์รอง เน้นการเตือน ลำเลียงแผนภูมิ การเรียกข้อมูล ความอิ่มตัวไม่เพียงพอที่จะจับคู่กับเซจโดยไม่มีความขัดแย้งทางภาพ

### นิวทรัล

- **Natural Background** (`oklch(0.98 0.005 85)`): พื้นหลังหน้า สีขาวเทพูปอบอุ่น—ไม่ปราศจากเชื้อ ไม่ครีม
- **Surface / Card** (`oklch(1 0 0)`): สีขาวบริสุทธิ์สำหรับพื้นผิวที่ยกขึ้น (การ์ด โมดอล) สร้างลำดับชั้นที่ชัดเจนโดยไม่มีความคมชัดที่รุนแรง
- **Secondary Surface** (`oklch(0.92 0.03 85)`): พื้นผิวเป็นสีอบอุ่นซ้อมสำหรับเนื้อหาที่จัดกลุ่ม พื้นหลังข้อมูลเข้า ส่วนที่มีการเน้นน้อย
- **Muted Surface** (`oklch(0.95 0.01 85)`): แม้แต่ยาง สำหรับตัวแบ่ง สถานะที่ปิดใช้งาน พื้นหลังการเน้นต่ำ
- **Text / Foreground** (`oklch(0.25 0.02 50)`): ข้อความเนื้อหา หมึกหลัก มืดเพียงพอสำหรับความคมชัด 7:1 ต่อพื้นหลัง (มาตรฐาน WCAG AAA)
- **Muted Text** (`oklch(0.40 0.02 50)`): ข้อความการเน้นต่อ ป้ายกำกับ คำใจ รักษาความคมชัด 7:1 เต็มไปด้วย WCAG AAA มาตรฐาน
- **Border / Divider** (`oklch(0.88 0.02 85)`): เส้น ตัวแบ่ง ขอบเขตที่ละเอียดอ่อน

### ความหมาย

- **Success** (`oklch(0.6 0.15 145)`): ยืนยัน ความสำเร็จ สถานะเชิงบวก ใช้เฉดสีเซจเดียวกันเพื่อความสม่ำเสมอทางภาพ
- **Warning** (`oklch(0.75 0.15 85)`): การเตือน เตือน สถานะรอดำเนินการ ตระหนักถึงสมาชิกแอมเบอร์ความเบา
- **Destructive** (`oklch(0.55 0.2 25)`): การลบ ยกเลิก ข้อผิดพลาด ความอิ่มตัวสูงแดง-น้ำตาลเพื่อความรุนแรงที่ไม่อยากลืม
- **Info** (`oklch(0.6 0.1 230)`): ข้อความข้อมูล ข้อมูลรอง สีน้ำเงินเย็นเพื่อความแตกต่างจากลำเลียงที่อบอุ่น

### กฎชื่อ

**กฎการปฏิเสธ** การเน้นหลัก (Fresh Sage) ถูกใช้ใน ≤10% ของหน้าจอใด ๆ ความเข้มแข็งของมันมาจากการขาดแคลน เมื่อทุกอย่างเป็นสีเขียว ไม่มีอะไรเลย นี่ใช้ลำดับชั้นโดยไม่มีเสียงรบกวน

**กฎ Warm Neutral** พื้นหลังและขอบเขตที่เป็นกลางทั้งหมดเป็นสีอบอุ่น ไม่เคยเป็นสีเทาบริสุทธิ์ นี่ช่วยให้อินเตอร์เฟซอ่อนลงและป้องกันความรู้สึกที่ "เย็น องค์กร" ซึ่งปฏิเสธการแบรนด์น่ารักทำงานต่อต้าน

## การพิมพ์

**Display Font:** Prompt (พร้อม Geist fallback sans-serif)  
**Body Font:** Prompt (ตระหนักถึงสมาชิก น้ำหนักต่าง ๆ)  
**Mono Font:** Geist Mono (สำหรับโค้ด ข้อมูล เนื้อหาที่มีโครงสร้าง)

**ลักษณะ:** Prompt คือ sans-serif เรขาคณิตที่เหมาะสำหรับไทยพร้อมสัดส่วนเท่า ๆ กันและความสามารถในการอ่านที่ดีในขนาดเล็ก ระบบใช้ความคมชัดน้ำหนัก (400–600) แทนตระหนักถึงสมาชิกสหายหลายคนเพื่อรักษาความสอดคล้อง คู่นี้เรียบง่าย มืออาชีพ และเคารพภาษาไทยเป็นสาขาการออกแบบชั้นแรก ไม่ใช่สิ่งที่พิจารณาภายหลัง

### ลำดับชั้น

- **Display** (600, clamp(2rem, 5vw, 3.5rem), 1.1): หัวเรื่องหน้าชาวไร่ ชื่อเรื่องส่วนหลัก ใช้อย่างหนัก ไม่เคยมากกว่าหนึ่งครั้งต่อหน้าจอ
- **Headline** (600, clamp(1.5rem, 4vw, 2.25rem), 1.2): ชื่อเรื่องส่วนหลัก ชื่อเรื่องโมดอล ชื่อเรื่องการ์ดแดชบอร์ด สัญญาณความสำคัญและโครงสร้าง
- **Title** (600, 1.25rem, 1.3): ชื่อเรื่องหมวดย่อย ชื่อฟีเจอร์ ป้ายกำกับเนื้อหาที่จัดกลุ่ม
- **Body** (400, 1rem, 1.6): ข้อความหลัก รายละเอียด รายชื่อข้อมูล คำแนะนำแบบฟอร์ม ขีด จำกัด ที่ ~65–75ch ความยาวเส้นเพื่อรักษาความสามารถในการอ่าน
- **Label** (500, 0.875rem, 1.4): ป้ายกำกับแบบฟอร์ม ป้ายกำกับ ชิป รายการนำทาง ข้อความ UI ขนาดเล็ก ปิดตัวเล็กน้อยมากกว่าเนื้อหาเพื่อความแตกต่าง
- **Mono** (400, 0.875rem, 1.6): คีย์ API รหัสธุรกรรม ข้อมูลดีบัก ข้อมูลที่มีโครงสร้าง ไม่ใช้ในข้อความเนื้อหา

### กฎชื่อ

**กฎความชัดเจน** ข้อความเนื้อหาต้องมีขนาดขั้นต่ำ 1rem / 16px และไม่เคยเบากว่า 400 ไม่มีข้อความเนื้อหาตัวพิมพ์ใหญ่ทั้งหมด ป้ายกำกับและข้อความขนาดเล็กมีขนาดขั้นต่ำ 0.875rem และน้ำหนัก 500 หรือสูงกว่า สิ่งนี้รับประกันความสามารถในการอ่านภายใต้การส่องสว่างของพื้นดินร้านโดยทั่วไป

**กฎ Thai-First** มาตราส่วนการพิมพ์และความสูงของบรรทัดจะปรับให้เหมาะสำหรับตัวชี้วัดอักขระภาษาไทย (ความสูงของ x ของฟอนต์ Prompt และระยะห่าง) ภาษาอังกฤษและภาษาไทยใช้มาตราส่วนเดียวกัน ไม่ใช่ความชันแยก

## ยก

ระบบนี้ใช้ **การจัดชั้นโทน** ไม่ใช่เงาลด ความลึกถูกถ่ายทำผ่านการเปลี่ยนแปลงสีพื้นหลังที่ละเอียดอ่อนและขอบเขต สร้างความรู้สึกที่เรียบร้อยมืออาชีพมากกว่าความสวยงามของ "ยกขึ้น" หรือ "บัญชี"

พื้นผิวเพิ่มขึ้นตามลำดับนี้:
- Background: `oklch(0.98 0.005 85)`
- พื้นผิวรอง (ส่วนสล่อม): `oklch(0.92 0.03 85)`
- Card / Popover / Modal: `oklch(1 0 0)` (สีขาวบริสุทธิ์)

โฟกัสและเงื่อนไขวางเมาส์ใช้สีวงแหวน (Fresh Sage ความอิ่มตัวลด) เป็นเค้าร่างหรือเรืองแสง ไม่เคยเงา

**กฎชื่อ**

**กฎแบบ Flat-by-Default** พื้นผิวแบนเมื่ออยู่นิ่ง ไม่มีเงา ไม่มีเบลอ ไม่มีเอฟเฟกต์ drop-shadow ขอบเขตและโทนพื้นหลังส่งต่อลำดับชั้น นี่เก็บอินเตอร์เฟซไว้ที่สั้น ๆ และลดเสียงรบกวนทางภาพ—เหมาะสำหรับพื้นที่ทำงานที่เข้มข้นโฟกัส

## องค์ประกอบ

องค์ประกอบทั้งหมดใช้ Radix UI เป็นชั้นดั้งเดิม ต่อไปนี้อธิบายการรักษาทางภาพและพฤติกรรม

### ปุ่ม

- **Shape:** มุมปัดที่ -md radius (0.625rem) สำหรับลักษณะที่เรียบร้อย ไม่ปัดเต็มที่
- **Primary:** Background Fresh Sage (`oklch(0.55 0.15 145)`) ข้อความสีขาว (`oklch(0.98 0.005 85)`) Padding 10px แนวนอน / 12px แนวตั้ง ใช้สำหรับการกระทำหลักบนหน้าจอ (บันทึก ส่ง ยืนยัน)
- **Hover / Active:** Background มืดขึ้นเล็กน้อยถึง `oklch(0.50 0.14 145)` ไม่มีเงาหรือแปลง พึ่งพาการเปลี่ยนสีและเค้าร่างโฟกัส
- **Secondary:** Background Secondary Surface (`oklch(0.92 0.03 85)`) ข้อความมืด (`oklch(0.25 0.02 50)`) สำหรับการกระทำที่มีความสำคัญน้อยกว่า
- **Ghost:** พื้นหลังโปร่งใส สีข้อความตรงกับอดีต ขอบเขตทางเลือกวางเมาส์ ใช้สำหรับยกเลิก ย้อนกลับ หรือการกระทำลำดับต่ำ

### การ์ด

- **Corner Style:** 0.75rem radius (rounded-lg) สอดคล้องกับส่วนที่เหลือของอินเตอร์เฟซ ไม่สุดขั้ว
- **Background:** สีขาวบริสุทธิ์ (`oklch(1 0 0)`) สำหรับเนื้อหาที่ยกขึ้น ไม่มีเงา พึ่งพาการเปลี่ยนโทนจากพื้นหลังหน้าจอ
- **Border:** ทางเลือก 1px ขอบที่ `oklch(0.88 0.02 85)` สำหรับคำจำกัดความที่ละเอียดอ่อนในเค้าโครงที่หนาแน่น
- **Internal Padding:** 16px (1rem) มาตรฐาน ระยะห่างที่ตึงขึ้น (8px) ในตารางข้อมูลหรือกริดที่หนาแน่น
- **Shadow Strategy:** ไม่มี การจัดชั้นโทนแบบแบน

### อินพุต

- **Style:** Background Secondary Surface (`oklch(0.92 0.01 85)`) 1px ขอบที่ `oklch(0.88 0.02 85)` 0.625rem radius
- **Focus:** Ring outline ใน Fresh Sage (ตัวแปร `--ring`) 2px inset หรือ outline (ลักษณะการกำหนดของเบราว์เซอร์ใช้ได้)
- **Error:** สีข้อความเปลี่ยนเป็นสีแดงที่ทำให้เสียหาย ทางเลือกขอบแดงหรือโทนพื้นหลังหากต้องการความแตกต่าง
- **Disabled:** Background Muted (`oklch(0.95 0.01 85)`) สีข้อความอยากย่อย pointer-events: ไม่มี
- **Placeholder:** ความต้องการความคมชัดเดียวกันกับข้อความเนื้อหา (ขั้นต่ำ 7:1) ไม่เคยสีเทาอ่อนบริสุทธิ์

### นำทาง / Sidebar

- **Style:** Background โทนอบอุ่น (`oklch(0.98 0.01 145)`) ชิ่วอบอุ่นกว่าพื้นหลังหน้าจอเล็กน้อยเพื่อสร้างคอนเทนเนอร์ที่กำหนด
- **Active Item:** พื้นหลังเซจ ข้อความสีขาว ปัดให้ตรงกับปุ่มรัศมี
- **Hover:** ยกพื้นหลังหรือเปลี่ยนสีข้อความที่ละเอียดอ่อน ไม่มีไฮไลท์ที่รุนแรง
- **Icons:** Lucide React ขนาดที่ 20–24px สีสืบทอดจากสีข้อความหรือสถานะใช้งาน

### บทสนทนา / Modals

- **Background:** Backdrop เป็นการวางซ้อนที่โปร่งใสบางส่วน (เช่น `rgba(0, 0, 0, 0.5)`) ที่ดัชนี z สูง แผงโมดอลเป็นสีขาวบริสุทธิ์ เงามดแบบ 1–2px ขอบหากจำเป็นสำหรับคำจำกัดความ
- **Padding:** 24px / 32px ขึ้นอยู่กับเนื้อหา

## ควร และ ไม่ควร

### ควร

- **ใช้ Fresh Sage (primary) น้อยกว่า 10% ของหน้าจอ** ความเข้มแข็งของมันมาจากการขาดแคลน เมื่อทุกอย่างเป็นสีเขียว ไม่มีอะไร
- **ให้ข้อความทั้งหมด (เนื้อหาและป้ายกำกับ) มี 1rem / 0.875rem หรือใหญ่กว่า** ข้อความที่เล็กกว่านี้อ่านไม่ได้ภายใต้การส่องสว่างทั่วไป ไม่มีข้อความเนื้อหา 12px
- **รักษาความคมชัด 7:1 ระหว่างข้อความเนื้อหาและพื้นหลัง** นี่คือ WCAG AAA ตรวจสอบมัน นี่ไม่ลดราคา
- **ใช้นิวทรัลอบอุ่น (ไม่เคยสีเทาบริสุทธิ์)** สีเทาทั้งหมดในระบบมีโทนหมายถึงเฉดสีเซจหรือสีอบอุ่น นี่ช่วยให้เครื่องมือมีความนุ่มนวลและป้องกัน "เย็นขององค์กร"
- **ชอบขอบเขตและการเปลี่ยนโทนมากกว่าเงา** ระบบแบนตามค่าเริ่มต้น ความลึกมาจากสี ไม่ใช่การยก
- **เคารพภาษาไทยในการตัดสินใจพิมพ์และระยะห่างทั้งหมด** Prompt เป็นฟอนต์เจ้าของไทยด้วยเหตุผลที่ดี อย่าปะแนวคาดหวังของภาษาอังกฤษบนข้อความไทย
- **ทดสอบความคมชัดของสีด้วยข้อความจริงภายใต้การส่องสว่างจริงของผู้ใช้** พื้นดินร้านที่ 9am แตกต่างจากสำนักงานที่มืด ความคมชัดมีความสำคัญ

### ไม่ควร

- **ไม่ใช้แบรนด์สัตว์เลี้ยงน่ารัก** ไม่มีมาสคอต ไม่มีอีโมจิที่เล่นสนุก ไม่มีสำเนาพูดอย่างเด็ก ไม่มีสีพาสเทล นี่คือเครื่องมือธุรกิจที่เคารพศิลปะแห่งการอาบน้ำตัดขนสัตว์เลี้ยง
- **ไม่ผสมพื้นหลังนิวทรัลสีเทาด้วยลำเลียงเป็นสีเทิม** สีเทาฆ่าความอบอุ่น นิวทรัลเป็นสีเทิม + เซจ + แอมเบอร์ = ฟิลม์เรียบร้อย สีเทา + เซจ + แอมเบอร์ = ความสับสนทางภาพ
- **ไม่ใช้รัศมีปัดที่มากกว่า 1.25rem (rounded-xl)** อินเตอร์เฟซเป็นมืออาชีพ ไม่ใช่คล้ายของเล่น ด้วย 1.25rem สูงสุดสำหรับมุมที่ผ่อนคลายมากที่สุด (ปุ่มขนาดใหญ่ โมดอล)
- **ไม่ปรับขนาดของการพิมพ์ต่ำกว่า 0.875rem สำหรับองค์ประกอบปฏิสัมพันธ์** ปุ่ม ป้ายกำกับ และข้อความแบบฟอร์มต้องอ่านได้ 0.875rem เป็นพื้น
- **อย่าพึ่งพาสีเพียงอย่างเดียวเพื่อส่งต่อสถานะ** ข้อความสีแดงบนพื้นหลังสีขาวนั้นดี ข้อความสีแดงบนพื้นหลังสีแดงไม่ใช่ เสมอจับคู่สีกับรูปร่าง ขอบเขต ไอคอน หรือข้อความ
- **ไม่เพิ่มเงา drop หรือเอฟเฟกต์เบลอประดับ** ระบบแบน เงาเลอะเลือนความชัดเจน หากคุณต้องการเน้น ให้ใช้ขอบเขต การเปลี่ยนพื้นหลัง หรือสีวงแหวน
- **อย่าละเลย WCAG 2.1 AAA มาตรฐานการเข้าถึงได้** นี่สร้างจากจุดเริ่มต้น ความคมชัด การนำทางคีย์บอร์ด การเคลื่อนไหวที่ลด—สิ่งเหล่านี้ไม่ใช่สิ่งที่ดี
- **ไม่ใช้ Warm Amber (accent) เป็นสีพื้นหลังสำหรับข้อความเนื้อหา** ความคมชัดล้มเหลว แอมเบอร์สำหรับไฮไลท์ ไอคอน และลำเลียงขนาดเล็กเท่านั้น
